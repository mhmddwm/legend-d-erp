# PUR-001 — Purchase Request Revision Management

## ما تم تنفيذه
- إضافة مراجعات Rev 0 / Rev 1 / Rev 2 لطلبات الشراء.
- عند تعديل طلب شراء محفوظ يتم تسجيل سبب التعديل والتغييرات.
- حفظ سجل المراجعات في localStorage باسم `purchase_request_revisions`.
- إظهار رقم المراجعة داخل قائمة طلبات الشراء.
- إضافة زر `السجل` لعرض تاريخ المراجعات.
- عند تعديل طلب شراء مرتبط بـ RFQ مفتوح يتم تعليم RFQ بأنه `يحتاج تحديث العرض`.

## طريقة الاختبار
1. افتح شاشة طلب شراء.
2. أنشئ طلب شراء جديد واحفظه.
3. افتح الطلب من قائمة الطلبات.
4. عدّل كمية أو أضف منتجًا واحفظ.
5. يظهر طلب سبب التعديل ثم يتم إنشاء Rev جديد.
6. اضغط زر `السجل` في قائمة الطلبات لمراجعة التغييرات.
