# LEGEND D Localization & Multilingual Engine

## الهدف
دعم تعدد اللغات كجزء من نواة المنصة، وليس كترجمة سطحية للواجهة فقط.

## المبادئ
- اللغة العربية تعمل باتجاه RTL.
- اللغة الإنجليزية تعمل باتجاه LTR.
- لا تكتب النصوص داخل الكود مباشرة في المراحل القادمة؛ استخدم مفاتيح ترجمة مركزية.
- بيانات الأعمال المهمة مثل اسم الحساب، المنتج، التصنيف، الوحدة، المورد، العميل يجب أن تدعم أكثر من لغة.

## شكل تخزين البيانات متعددة اللغات

```json
{
  "name": {
    "ar": "حبة",
    "en": "Piece"
  }
}
```

## ما تم في هذه المرحلة
- إضافة `frontend/i18n.js` كمحرك ترجمة أولي.
- ربط زر تغيير اللغة بالمحرك.
- ترجمة النصوص الثابتة والـ placeholders والعناصر الديناميكية قدر الإمكان.
- إضافة `backend/app/core/localization` كمحرك Backend مبدئي.
- إضافة API مبدئي للغات: `/api/localization/languages`.

## المرحلة القادمة
- نقل النصوص إلى ملفات JSON منفصلة لكل لغة.
- إضافة جداول قاعدة بيانات للترجمات.
- جعل أسماء الحسابات والمنتجات والوحدات والموردين متعددة اللغات فعليًا.
- ربط قوالب الطباعة والإيميل والواتساب باللغة المختارة.
